OpenMile — OBD-II Data Platform

Overview

OpenMile is a startup initiative exploring how drivers can monetize their vehicle data. The project investigates secure ingestion of OBD-II telemetry, data packaging, and resale pathways through intelligent APIs and app-driven user experiences. The early work included prototyping, mobile development experiments, and building out foundational data pipelines.

What I Built

  • Designed early prototypes using iOS + Bluetooth OBD-II readers.
  • Built cross-platform UI experiments using Tauri, Neutralino, and JavaScript frameworks.
  • Created telemetry ingestion and transformation concepts for vehicle data packaging.
  • Maintained structured product journals: OpenMileProtoV1, business logs, and dev notes.
  • Explored user-facing app flows for scanning, syncing, and monetizing driving data securely.

Key Outcomes

  • Established an early-stage technical and business model for a data-monetization platform.
  • Created a repeatable prototype workflow for testing sensors, APIs, and mobile apps.
  • Captured repeatable documentation patterns for ongoing startup development.

Tools & Technologies

  • OBD-II Bluetooth Scanners
  • iOS Development / Xcode
  • Tauri, Neutralino, JavaScript
  • REST APIs
  • Product journaling and version tracking
